# dantuck.com Writing Style

## Voice

- First-person, grounded, direct
- No contractions: "it is" not "it's", "do not" not "don't", "I have" not "I've"
- No hedging or filler qualifiers ("somehow", "sort of", "kind of")
- Honest and personal without being self-indulgent

## Sentence style

- Short, punchy sentences preferred. One idea per sentence.
- Vary length — a short sentence after a longer one creates emphasis. "That sounds small. It is not small."
- Em-dashes for parenthetical asides — use sparingly
- Avoid run-on clauses strung together with repeated "and" and "but"

## Structure

- Opening: 1-2 punchy sentences before the first section, or jump straight to the first heading — no long preamble
- Section headers: sentence case, specific and descriptive, sometimes oblique ("The problem with reading", "What has actually changed")
- No generic headers like "Introduction" or "Conclusion"
- Horizontal rules (`---`) between major topic shifts within a section
- No bold call-out labels like "**Bottom line:**" — write the closing as a plain sentence

## Links

- Inline contextual links, never "click here"
- Use `<a href="..." target="_blank" rel="noopener noreferrer">text</a>` HTML syntax, not markdown links

## Tags taxonomy

Lowercase, hyphenated. 2-4 tags per article. Existing tags:
`reading`, `tools`, `accessibility`, `claude-code`, `ai`, `developer-tools`, `workflow`, `sci-fi`, `movies`, `backup`, `self-hosting`, `homelab`
